Scaled defaults: embed_dim=512, num_layers=12.

2. Architecture Overview

Input bytes (0–255) │ ▼ ┌─────────────────────────────────────┐ │ nn.Embedding(257, D) │ vocab: 0-255 = bytes, 256 = PAD/SEP └─────────────────────────────────────┘ │ shape: (B, T, D) ▼ ┌─────────────────────────────────────┐ │ EntropyPatchEncoder │ Conv1d with optional entropy boundaries │ └─ BytePosEmbedding injected here │ sinusoidal byte-offset embs summed into patches └─────────────────────────────────────┘ │ shape: (B, T_p, D) ▼ ┌─────────────────────────────────────┐ │ N × TransformerLayer │ pre-norm, RoPE, causal self-attention │ (gradient checkpointing optional) │ FlashAttention via is_causal=True └─────────────────────────────────────┘ │ shape: (B, T_p, D) ▼ ┌─────────────────────────────────────┐ │ OverlapAddDecoder │ learned window + fold-based overlap-add │ (replaces ConvTranspose1d) │ eliminates patch-boundary artefacts └─────────────────────────────────────┘ │ shape: (B, T_r, D) ▼ ┌─────────────────────────────────────┐ │ nn.Linear(D, 257) │ byte prediction head └─────────────────────────────────────┘ │ ▼ Next-byte logits (+ CFG scale at inference) 

CFG training protocol: 10 % of samples have their caption replaced with an empty byte sequence. At inference, logits are computed twice — once with and once without the caption — and blended:

logits_guided = logits_uncond + guidance_scale * (logits_cond - logits_uncond)

============================================================================= # Rotary Positional Embeddings — unchanged from V1 # ============================================================================= class RotaryEmbedding(nn.Module): def __init__(self, dim: int, max_position_embeddings: int = 32768, base: float = 500000.0, device: Optional[str] = None): super().__init__() self.dim = dim self.max_position_embeddings = max_position_embeddings self.base = base inv_freq = 1.0 / (self.base ** (torch.arange(0, self.dim, 2).float() / self.dim)) self.register_buffer("inv_freq", inv_freq, persistent=False) self._build_cache(max_position_embeddings, self.inv_freq.device, torch.get_default_dtype()) def _build_cache(self, seq_len: int, device, dtype): self.max_seq_len_cached = seq_len t = torch.arange(seq_len, device=device, dtype=self.inv_freq.dtype) freqs = torch.outer(t, self.inv_freq) emb = torch.cat((freqs, freqs), dim=-1) self.register_buffer("cos_cached", emb.cos().to(dtype), persistent=False) self.register_buffer("sin_cached", emb.sin().to(dtype), persistent=False) def forward(self, x: torch.Tensor, seq_len: int) -> Tuple[torch.Tensor, torch.Tensor]: if seq_len > self.max_seq_len_cached: self._build_cache(seq_len, x.device, x.dtype) return ( self.cos_cached[:seq_len].to(dtype=x.dtype), self.sin_cached[:seq_len].to(dtype=x.dtype), ) def rotate_half(x: torch.Tensor) -> torch.Tensor: x1, x2 = x[..., :x.shape[-1] // 2], x[..., x.shape[-1] // 2:] return torch.cat((-x2, x1), dim=-1) def apply_rotary_pos_emb(q, k, cos, sin): cos = cos.unsqueeze(0).unsqueeze(0) sin = sin.unsqueeze(0).unsqueeze(0) q = (q * cos) + (rotate_half(q) * sin) k = (k * cos) + (rotate_half(k) * sin) return q, k # ============================================================================= # NEW V2: BytePosEmbedding — absolute byte-offset embeddings inside patches # ============================================================================= class BytePosEmbedding(nn.Module): """ Injects absolute byte-position information into each patch representation. Motivation (from V1 limitations): After patch encoding, RoPE only knows *patch* indices, not where in the raw byte stream each patch came from. For JPEG specifically, absolute byte offset matters: the SOI marker (FFD8) is always at byte 0, quantisation tables at a known offset, entropy-coded scan data starting at a predictable position. Giving the model this grounding helps it reconstruct structured byte sequences. Implementation: For a batch of patches at byte offsets [o_0, o_1, ..., o_{T_p-1}], we compute a sinusoidal embedding of each offset (same formula as the original Transformer PE, but over byte offsets rather than sequence positions) and add it to the patch features. The embedding dimension matches embed_dim. Offsets are the *start* byte index of each patch: offset[i] = i * patch_stride (fixed-stride case) For entropy patching, offsets are accumulated from variable patch widths. """ def __init__(self, embed_dim: int, max_seq_len: int = 32768): super().__init__() self.embed_dim = embed_dim # Pre-compute sinusoidal table up to max_seq_len byte positions pe = torch.zeros(max_seq_len, embed_dim) position = torch.arange(0, max_seq_len, dtype=torch.float).unsqueeze(1) div_term = torch.exp( torch.arange(0, embed_dim, 2).float() * (-math.log(10000.0) / embed_dim) ) pe[:, 0::2] = torch.sin(position * div_term) pe[:, 1::2] = torch.cos(position * div_term[:embed_dim // 2]) self.register_buffer("pe", pe, persistent=False) # (max_seq_len, D) def forward(self, patches: torch.Tensor, byte_offsets: torch.Tensor) -> torch.Tensor: """ Args: patches: (B, T_p, D) — patch features byte_offsets: (T_p,) long — start byte index of each patch Returns: (B, T_p, D) — patches with byte-position signal added """ # Clamp to table size offsets = byte_offsets.clamp(max=self.pe.shape[0] - 1) pos_emb = self.pe[offsets] # (T_p, D) return patches + pos_emb.unsqueeze(0) # broadcast over batch # ============================================================================= # NEW V2: EntropyPatchEncoder — fixed-stride OR entropy-based patching # ============================================================================= class EntropyPatchEncoder(nn.Module): """ Encodes raw byte embeddings into patches, optionally using entropy-based variable-length boundaries (BLT-style). Fixed-stride mode (use_entropy=False, default): Identical to V1 BytePatchEncoder — Conv1d with kernel=16, stride=8. Fast, simple, well-understood. Entropy-based mode (use_entropy=True): Estimates local byte entropy using a sliding window and places patch boundaries where normalised entropy exceeds a threshold. Low-entropy regions (flat sky, uniform backgrounds) get merged into larger patches, spending fewer Transformer tokens on redundant content. High-entropy regions (edges, text, detail) get finer patches. Implementation note: True BLT uses a learned boundary predictor. Our approximation uses the empirical byte distribution in a local window, which is cheaper but still captures the key intuition. For production use, a small 1D CNN boundary predictor trained jointly would be more accurate. After patching (both modes), BytePosEmbedding is optionally injected. """ def __init__(self, config: ModelConfig): super().__init__() self.config = config D = config.embed_dim # Strided conv (used in both modes — as fallback and in fixed mode) self.conv = nn.Conv1d( in_channels=D, out_channels=D, kernel_size=config.patch_kernel, stride=config.patch_stride, padding=0, bias=False, ) self.norm = nn.RMSNorm(D) # Byte position embeddings (V2) if config.use_byte_pos_embeddings: self.byte_pos_emb = BytePosEmbedding(D, config.max_seq_len) else: self.byte_pos_emb = None # For entropy patching: a small projection to aggregate variable-width patches if config.use_entropy_patching: self.patch_proj = nn.Linear(D, D, bias=False) def forward( self, x: torch.Tensor # (B, T, D) ) -> Tuple[torch.Tensor, torch.Tensor]: """ Returns: patches: (B, T_p, D) byte_offsets: (T_p,) long — start byte of each patch (for BytePosEmbedding) """ B, T, D = x.shape if self.config.use_entropy_patching: patches, byte_offsets = self._entropy_patch(x) else: patches, byte_offsets = self._fixed_patch(x) patches = self.norm(patches) if self.byte_pos_emb is not None: patches = self.byte_pos_emb(patches, byte_offsets) return patches, byte_offsets def _fixed_patch(self, x: torch.Tensor) -> Tuple[torch.Tensor, torch.Tensor]: """Standard strided-conv patching from V1.""" B, T, D = x.shape x_t = x.transpose(1, 2) # (B, D, T) out = self.conv(x_t).transpose(1, 2) # (B, T_p, D) T_p = out.shape[1] byte_offsets = torch.arange(T_p, device=x.device) * self.config.patch_stride return out, byte_offsets def _entropy_patch(self, x: torch.Tensor) -> Tuple[torch.Tensor, torch.Tensor]: """ Variable-length patching using local byte entropy estimates. For each position in the sequence, we estimate local entropy over a small window and decide whether to start a new patch. Patches are then mean-pooled and projected. This runs on the embedding dimension directly (not raw bytes), using the L2 norm of embedding differences as a proxy for local complexity. """ B, T, D = x.shape device = x.device min_p = self.config.entropy_patch_min max_p = self.config.entropy_patch_max threshold = self.config.entropy_threshold # Compute local complexity proxy: normalised L2 of consecutive differences # Shape: (B, T-1) diffs = (x[:, 1:] - x[:, :-1]).norm(dim=-1) # Normalise per sequence diffs = diffs / (diffs.max(dim=-1, keepdim=True).values + 1e-8) # Build patch boundaries for the first item in the batch (B=1 for simplicity; # for B>1 we fall back to fixed-stride to avoid ragged tensors) if B > 1: # Variable-length patches require ragged tensors; fall back to fixed-stride # in batched training. Entropy patching is most useful at inference. return self._fixed_patch(x) boundaries = [0] pos = 0 while pos < T: # Advance by at least min_p positions next_pos = min(pos + min_p, T) # Check for high-complexity boundary within [next_pos, pos+max_p] while next_pos < min(pos + max_p, T): if diffs[0, next_pos - 1].item() > threshold: break next_pos += 1 boundaries.append(next_pos) pos = next_pos # Pool each patch patch_list = [] offsets = [] for i in range(len(boundaries) - 1): start, end = boundaries[i], boundaries[i + 1] patch = x[0, start:end].mean(dim=0, keepdim=True) # (1, D) patch_list.append(patch) offsets.append(start) patches = self.patch_proj( torch.stack(patch_list, dim=0) # (T_p, 1, D) — needs squeeze .squeeze(1) # (T_p, D) .unsqueeze(0) # (1, T_p, D) ) byte_offsets = torch.tensor(offsets, dtype=torch.long, device=device) return patches, byte_offsets def output_length(self, input_length: int) -> int: """Approximate output length for fixed-stride mode.""" return (input_length - self.config.patch_kernel) // self.config.patch_stride + 1 # ============================================================================= # NEW V2: OverlapAddDecoder — replaces ConvTranspose1d # ============================================================================= class OverlapAddDecoder(nn.Module): """ Learned overlap-add upsampling decoder. Problem with V1 ConvTranspose1d: Transposed convolution upsampling produces checkerboard / boundary artefacts because the same kernel is applied independently at each stride position, creating discontinuities at patch edges. For high- frequency JPEG data (entropy-coded scan segments), these artefacts are visible as periodic noise patterns in generated bytes. Overlap-add solution: Each patch representation is projected to a frame of length `kernel` via a learned linear projection, then the frames are overlapped and added with a learned smooth window (similar to WOLA in audio synthesis). The window enforces smooth transitions between patches. The output length formula is the same as ConvTranspose1d: T_reconstructed = (T_p - 1) * stride + kernel Parameter count is comparable to ConvTranspose1d for the same kernel/stride. """ def __init__(self, embed_dim: int, kernel: int = 16, stride: int = 8): super().__init__() self.embed_dim = embed_dim self.kernel = kernel self.stride = stride # Project each patch vector to a frame of length kernel * embed_dim self.frame_proj = nn.Linear(embed_dim, kernel * embed_dim, bias=False) # Learned synthesis window — initialised to Hann window for smooth default window_init = torch.hann_window(kernel, periodic=False) self.window = nn.Parameter(window_init.unsqueeze(-1)) # (kernel, 1) self.norm = nn.RMSNorm(embed_dim) def forward(self, patches: torch.Tensor) -> torch.Tensor: """ Args: patches: (B, T_p, D) Returns: (B, T_reconstructed, D) """ B, T_p, D = patches.shape K = self.kernel S = self.stride # Project each patch to a frame: (B, T_p, K*D) frames = self.frame_proj(patches) # (B, T_p, K*D) frames = frames.view(B, T_p, K, D) # (B, T_p, K, D) # Apply window: (K, 1) broadcasts over D window = torch.sigmoid(self.window) # learnable smooth window frames = frames * window.unsqueeze(0).unsqueeze(0) # (B, T_p, K, D) # Overlap-add via fold # fold expects (B, K*D, T_p) T_out = (T_p - 1) * S + K frames_flat = frames.permute(0, 2, 3, 1) # (B, K, D, T_p) frames_flat = frames_flat.reshape(B, K * D, T_p) # (B, K*D, T_p) # Use F.fold to perform the overlap-add output = F.fold( frames_flat, output_size=(1, T_out), kernel_size=(1, K), stride=(1, S), ) # (B, K*D, 1, T_out) — no, wrong dim # F.fold works over 2D; for 1D we need a workaround: # Reshape as (B*D, K, T_p), fold to (B*D, 1, T_out), reshape back frames_bd = frames.permute(0, 3, 1, 2) # (B, D, T_p, K) frames_bd = frames_bd.reshape(B * D, T_p, K) # (B*D, T_p, K) frames_bd = frames_bd.transpose(1, 2) # (B*D, K, T_p) output = F.fold( frames_bd, output_size=(1, T_out), kernel_size=(1, K), stride=(1, S), ) # (B*D, 1, 1, T_out) output = output.squeeze(1).squeeze(1) # (B*D, T_out) # Normalise by overlap count (window sum at each position) # Build a ones-like input to get the count map ones = torch.ones(B * D, K, T_p, device=patches.device, dtype=patches.dtype) count = F.fold(ones, output_size=(1, T_out), kernel_size=(1, K), stride=(1, S)) count = count.squeeze(1).squeeze(1).clamp(min=1e-6) output = output / count # (B*D, T_out) output = output.view(B, D, T_out).transpose(1, 2) # (B, T_out, D) return self.norm(output) def output_length(self, patched_length: int) -> int: return (patched_length - 1) * self.stride + self.kernel # ============================================================================= # Self-Attention — FlashAttention compatible (unchanged from V1) # ============================================================================= class SelfAttention(nn.Module): def __init__(self, config: ModelConfig): super().__init__() self.num_heads = config.num_heads self.embed_dim = config.embed_dim self.head_dim = self.embed_dim // self.num_heads assert self.head_dim * self.num_heads == self.embed_dim self.qkv_proj = nn.Linear(self.embed_dim, 3 * self.embed_dim, bias=False) self.out_proj = nn.Linear(self.embed_dim, self.embed_dim, bias=False) self.rotary_emb = RotaryEmbedding( self.head_dim, max_position_embeddings=config.max_seq_len, base=config.rope_base, ) self.dropout_p = config.dropout def forward(self, x: torch.Tensor, key_padding_mask: Optional[torch.Tensor] = None) -> torch.Tensor: B, T, _ = x.shape qkv = self.qkv_proj(x) q, k, v = torch.chunk(qkv, 3, dim=-1) q = q.view(B, T, self.num_heads, self.head_dim).transpose(1, 2) k = k.view(B, T, self.num_heads, self.head_dim).transpose(1, 2) v = v.view(B, T, self.num_heads, self.head_dim).transpose(1, 2) cos, sin = self.rotary_emb(q, seq_len=T) q, k = apply_rotary_pos_emb(q, k, cos, sin) dropout_p = self.dropout_p if self.training else 0.0 if key_padding_mask is not None: additive_mask = torch.zeros(B, 1, 1, T, device=x.device, dtype=q.dtype) additive_mask = additive_mask.masked_fill( key_padding_mask[:, None, None, :], float('-inf') ) attn_out = F.scaled_dot_product_attention( q, k, v, attn_mask=additive_mask, dropout_p=dropout_p, is_causal=True ) else: attn_out = F.scaled_dot_product_attention( q, k, v, dropout_p=dropout_p, is_causal=True ) attn_out = attn_out.transpose(1, 2).contiguous().view(B, T, self.embed_dim) return self.out_proj(attn_out) # ============================================================================= # SwiGLU FFN + TransformerLayer (unchanged from V1, + checkpoint support) # ============================================================================= class SwiGLU(nn.Module): def __init__(self, embed_dim: int, ffn_dim: int): super().__init__() self.gate_proj = nn.Linear(embed_dim, ffn_dim, bias=False) self.up_proj = nn.Linear(embed_dim, ffn_dim, bias=False) self.down_proj = nn.Linear(ffn_dim, embed_dim, bias=False) def forward(self, x: torch.Tensor) -> torch.Tensor: return self.down_proj(F.silu(self.gate_proj(x)) * self.up_proj(x)) class TransformerLayer(nn.Module): def __init__(self, config: ModelConfig): super().__init__() ffn_dim = config.embed_dim * config.ffn_multiplier self.self_attn = SelfAttention(config) self.feed_forward = SwiGLU(config.embed_dim, ffn_dim) self.norm1 = nn.RMSNorm(config.embed_dim) self.norm2 = nn.RMSNorm(config.embed_dim) self.dropout = nn.Dropout(config.dropout) def forward(self, x: torch.Tensor, key_padding_mask: Optional[torch.Tensor] = None) -> torch.Tensor: residual = x x = self.norm1(x) x = residual + self.dropout(self.self_attn(x, key_padding_mask=key_padding_mask)) residual = x x = self.norm2(x) x = residual + self.dropout(self.feed_forward(x)) return x # ============================================================================= # Core Model: ByteTransformer V2 # ============================================================================= class ByteTransformer(nn.Module): """ Byte-level causal Transformer — V2. Key V2 additions: 1. EntropyPatchEncoder replaces BytePatchEncoder — same interface, adds optional entropy patching and always injects BytePosEmbedding. 2. OverlapAddDecoder replaces BytePatchDecoder — eliminates boundary artefacts. 3. Gradient checkpointing per TransformerLayer (config.gradient_checkpointing). 4. CFG support in generate(): pass guidance_scale > 1.0 for stronger text conditioning. Requires the model to have been trained with cfg_dropout_prob > 0. 5. Trim diagnostic in _compute_loss(). """ def __init__(self, config: ModelConfig): super().__init__() self.config = config self.embedding = nn.Embedding( config.vocab_size, config.embed_dim, padding_idx=config.pad_token_id ) self.patch_encoder = EntropyPatchEncoder(config) self.layers = nn.ModuleList([ TransformerLayer(config) for _ in range(config.num_layers) ]) self.patch_decoder = OverlapAddDecoder( config.embed_dim, config.patch_kernel, config.patch_stride ) self.head = nn.Linear(config.embed_dim, config.vocab_size, bias=False) self.head.weight = self.embedding.weight # weight tying self._trim_log_accumulator = [] # V2: trim diagnostic self._init_weights() def _init_weights(self): for module in self.modules(): if isinstance(module, nn.Linear): nn.init.normal_( module.weight, mean=0.0, std=0.02 / math.sqrt(2 * self.config.num_layers) ) if module.bias is not None: nn.init.zeros_(module.bias) elif isinstance(module, nn.Embedding): nn.init.normal_(module.weight, mean=0.0, std=0.02) def _run_layers(self, x: torch.Tensor, patch_padding_mask: Optional[torch.Tensor]) -> torch.Tensor: """Run transformer layers, optionally with gradient checkpointing.""" for layer in self.layers: if self.config.gradient_checkpointing and self.training: # gradient_checkpointing_func requires no kwargs — wrap in lambda def create_custom_forward(mod): def custom_forward(*inputs): return mod(inputs[0], key_padding_mask=inputs[1] if len(inputs) > 1 else None) return custom_forward mask_arg = patch_padding_mask if patch_padding_mask is not None \ else torch.zeros(x.shape[0], x.shape[1], dtype=torch.bool, device=x.device) x = checkpoint.checkpoint( create_custom_forward(layer), x, mask_arg, use_reentrant=False, ) else: x = layer(x, key_padding_mask=patch_padding_mask) return x def forward( self, x: torch.Tensor, # (B, T) long padding_mask: Optional[torch.Tensor] = None, # (B, T) bool targets: Optional[torch.Tensor] = None, # (B, T) long ) -> Tuple[torch.Tensor, Optional[torch.Tensor]]: B, T = x.shape # 1. Embed emb = self.embedding(x) # (B, T, D) # 2. Pad to minimum patch width if necessary if T < self.config.patch_kernel: pad_len = self.config.patch_kernel - T emb = F.pad(emb, (0, 0, 0, pad_len)) if padding_mask is not None: padding_mask = F.pad(padding_mask.float(), (0, pad_len), value=1.0).bool() # 3. Patch encode (returns byte_offsets for BytePosEmbedding) patched, byte_offsets = self.patch_encoder(emb) # (B, T_p, D), (T_p,) # 4. Patch-level padding mask patch_padding_mask = None if padding_mask is not None: patch_padding_mask = self._pool_padding_mask(padding_mask, patched.shape[1]) # 5. Transformer out = self._run_layers(patched, patch_padding_mask) # (B, T_p, D) # 6. Decode out = self.patch_decoder(out) # (B, T_r, D) # 7. Logits logits = self.head(out) # (B, T_r, V) # 8. Loss loss = None if targets is not None: loss = self._compute_loss(logits, targets) return logits, loss def _pool_padding_mask(self, byte_mask: torch.Tensor, patched_len: int) -> torch.Tensor: mask_float = byte_mask.float().unsqueeze(1) pooled = F.max_pool1d( mask_float, kernel_size=self.config.patch_kernel, stride=self.config.patch_stride, padding=0, ).squeeze(1) pooled = pooled[:, :patched_len] if pooled.shape[1] < patched_len: pooled = F.pad(pooled, (0, patched_len - pooled.shape[1]), value=1.0) return pooled.bool() def _compute_loss( self, logits: torch.Tensor, targets: torch.Tensor ) -> torch.Tensor: """ Next-byte prediction loss with trim diagnostic. V2 addition: logs the trim fraction (bytes silently dropped due to encode/decode rounding) every time it is called while self.training. Large trim values (> 5 %) indicate the patch encoder/decoder rounding is significant and should be investigated. 11. Known Limitations & Next Steps

Remaining limitations

Image quality at moderate scale. At 12 layers / 512 dimensions / 500k pairs the model will produce JPEG-structured bytes with recognisable macro-structure (colour statistics, rough spatial layout) but fine detail will be blurry or incorrect. Coherent photorealistic imagery is still hundreds of millions of pairs away. This is consistent with reported EvaByte results.

Entropy patching falls back to fixed-stride in batched training. The variable-length patch boundary varies per sequence, making batching require padding or ragged tensors. The current implementation falls back to fixed-stride when B > 1 in the encoder. A production solution would use a learned boundary predictor that outputs the same number of patches regardless of sequence content, or use dynamic batching with sequences of similar complexity.

CFG requires two forward passes per generation step. At 12 layers this roughly doubles inference time. For interactive applications, consider using CFG only for the first N steps (the ones that determine global structure) and turning it off for the tail.

FID is unreliable below ~1000 samples. At small evaluation budgets, use CLIP-score instead. FID is included for completeness and for when you scale up enough for it to be meaningful.

Suggested next steps in priority order

Scale data to 1M+ pairs — LAION-COCO via img2dataset is the single highest-leverage change for image quality.

Scale model to embed_dim=1024, num_layers=24 — straightforward; enable gradient checkpointing and accumulate gradients more aggressively.

Replace JPEG with a more compressible image format — JPEG at quality=50 and 128×128 reduces average image sequence length by ~4×, allowing higher effective throughput during early training. Upscale target resolution progressively.

Learned boundary predictor for entropy patching — a small 2-layer CNN trained jointly to output fixed-count patch boundaries would unlock true BLT-style efficiency gains in batched training.

Speculative decoding for faster inference — autoregressive byte-level generation is slow; a small draft model generating 4–8 byte candidates per step, verified by the full model, could give 3–5× speedup.

TinyByte Multimodal V2 — Tokenizer-free. Bytes are all you need.
